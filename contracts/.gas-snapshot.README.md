# Gas snapshots

Foundry writes per-test gas to `.gas-snapshot` after `forge snapshot`. Treat that
file as a CI-tracked baseline: it must be regenerated and committed whenever
contract changes intentionally move gas, and CI runs `forge snapshot --check`
to fail any unintended regression.

## Targets (Sprint 1, mock harness)

| Function | Budget |
|---|---|
| `createBounty` (no tags) | < 350k |
| `takeBounty` (human) | < 250k |
| `submitWork` | < 130k |
| `approveBounty` | < 200k |
| `autoApprove` | < 200k |
| `cancelBounty` (pre-take) | < 80k |

These are loose upper bounds — the real numbers depend on AC + USDC mocks. Real
fork measurements will be tighter; track them separately once fork tests run in CI.

## Regenerate

```bash
cd contracts
forge snapshot
git add .gas-snapshot
```

## Check (CI)

```bash
forge snapshot --check
```
